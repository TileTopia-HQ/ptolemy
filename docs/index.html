ptolemy-storage

  • Pluggable backend architecture

ptolemy-api

  • Axum 0.8 HTTP router
  • Tower middleware (tracing, CORS)
  • Typed JSON responses

ptolemy-cli

  • Clap-based CLI
  • ptolemy serve --bind 0.0.0.0:3000
  • Future: import, export, branch commands

Real-Time Collaboration Relay

Ptolemy includes an ephemeral room-based WebSocket relay at /ws/rooms/{room_id} for real-time viewer collaboration. Every JSON message sent by one participant is broadcast to all others in the same room. No messages are persisted — rooms are created on first connection and dropped when the last client disconnects.

📡 View Sync (Follow Mode)

  • Broadcast camera state (lat, lng, zoom, bearing, pitch)
  • Follower's viewer mirrors the leader's view in real-time
  • Throttled at 200ms, delta-only to minimise bandwidth

🖱️ Cursor Sharing

  • Share mouse position on the map between collaborators
  • See other users' cursors as coloured dots
  • Configurable throttle rate

👥 Presence & Chat

  • Track which users are online in a room
  • User name and colour assignment
  • Real-time text messaging within a room

Protocol (JSON over WebSocket)

// Connect to a room
ws://localhost:3000/ws/rooms/my-room

// Client → Server (relayed to all other clients)
{ "type": "Join",   "user_id": "u1", "user_name": "Alice", "asset_id": "my-room" }
{ "type": "Camera", "user_id": "u1", "latitude": 40.7, "longitude": -73.9, "zoom": 14, "bearing": 0, "pitch": 45 }
{ "type": "Cursor", "user_id": "u1", "latitude": 40.71, "longitude": -73.91 }
{ "type": "Chat",   "user_id": "u1", "user_name": "Alice", "message": "Look at this area" }
{ "type": "Leave",  "user_id": "u1", "asset_id": "my-room" }

Messages are opaque to the server — it relays any valid text frame to all other subscribers. The schema above is the convention used by ViewTopia's collaboration client.

Quick Start

# Clone and build
git clone https://github.com/TileTopia-HQ/ptolemy.git
cd ptolemy
cargo build --release

# Start with a PostGIS database
export DATABASE_URL="postgres://user:pass@localhost/ptolemy"
./target/release/ptolemy serve --bind 0.0.0.0:3000

# Check health
curl http://localhost:3000/health
