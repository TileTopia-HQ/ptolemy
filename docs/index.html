Features

🔀 Versioned Geodata

  • Git-like branching & merging for features
  • Full changeset history with diffs
  • Branch-level isolation for concurrent edits
  • Conflict detection & resolution
  • Time-travel queries (any revision)

🗄️ PostGIS Backend

  • PostgreSQL + PostGIS for spatial indexing
  • GeoJSON feature storage
  • Spatial queries (intersect, within, buffer)
  • Full ACID transactions
  • Connection pooling (sqlx)

🌐 REST API

  • Axum 0.8 async HTTP server
  • Dataset, branch, and feature CRUD
  • Commit endpoint with DiffOp operations
  • Branch merge with conflict reporting
  • Diff between any two changesets
  • Branch history (commit log)
  • Spatial queries (bbox, intersects, within)
  • MVT vector tile endpoint
  • Cursor-based pagination
  • Batch commit operations
  • Health checks & readiness probes

🤝 Collaboration

  • Multi-user concurrent editing
  • Named branches per team/task
  • Review & merge workflows
  • WebSocket real-time branch events
  • Three-way merge with conflict detection

📐 Geospatial Types

  • Point, LineString, Polygon, Multi-*
  • Arbitrary JSON properties per feature
  • Schema-on-read flexibility
  • UUID v7 for time-ordered IDs
  • WKB/GeoJSON interchange

🚀 Deployment

  • Single binary CLI (ptolemy serve)
  • Docker + docker-compose
  • ENV-based configuration
  • GeoJSON import/export
  • Full CLI: dataset, branch, commit, merge, log, diff
  • Prometheus metrics endpoint (/metrics)
  • Graceful shutdown (SIGTERM/SIGINT)
  • Configurable connection pool

🔐 Authentication

  • JWT authentication (PTOLEMY_JWT_SECRET)
  • Role-based access control (Admin/Editor/Viewer)
  • OIDC Single Sign-On (Keycloak, Auth0, etc.)
  • Token exchange & user info retrieval
  • Per-endpoint permission checks

📏 Schema & Topology

  • Dataset schema definitions (field types)
  • Geometry rules (allowed types, bounds, max vertices)
  • Topology rules (no overlap, no gaps, must connect)
  • Automated data quality reports
  • Auto-repair invalid geometries (ST_MakeValid)

📡 Webhooks & CDC

  • Webhook subscriptions per dataset
  • Event types: commit, merge, branch, schema changes
  • CDC event log with full payloads
  • Queryable event stream (pagination, filtering)
  • Custom event emission API

📊 Spatial Analytics

  • Buffer analysis (per-feature, configurable distance)
  • Union analysis (aggregate geometry)
  • DBSCAN spatial clustering
  • Anomaly detection (outliers, self-intersections)
  • Branch-level spatial statistics

🌍 OGC Compliance

  • OGC API - Features 1.0 Core
  • GeoJSON feature collections
  • Bbox filtering, pagination
  • Conformance declaration
  • Enterprise audit logging

🔒 Feature Locking

  • Pessimistic locking per feature/branch
  • Configurable lock duration & auto-expiry
  • Lock checks enforced on commit
  • Conflict prevention for high-contention edits
  • Lock listing & manual release

⏱️ Temporal Queries

  • Query features at any point in time
  • Time-travel across branch history
  • Historical state reconstruction
  • Audit-friendly snapshotting

📂 Data Catalog

  • Full-text search across datasets
  • Tag-based organization & filtering
  • Rich metadata (source, license, attribution)
  • Keyword indexing

🏢 Multi-Tenancy

  • Organization-level isolation
  • Member management & roles
  • Per-org dataset scoping
  • Rate limiting per IP (token bucket)

⚡ gRPC & Background Jobs

  • Binary protocol for bulk import/export
  • Protobuf message format (prost)
  • Scheduled lock cleanup & event pruning
  • Quality alert background checks
  • Conflict resolution API

🕸️ Geometric Networks

  • Graph topology: junctions + edges
  • Upstream/downstream/bidirectional trace
  • Dijkstra shortest path via recursive CTE
  • Connectivity analysis & isolated node detection
  • Utility network support

📏 Linear Referencing

  • Route creation with measured geometries
  • Point & linear events along routes
  • Locate point → measure (ST_LineLocatePoint)
  • Measure → sub-line extraction
  • ST_LocateAlong / ST_LocateBetween

🛰️ Raster & Imagery

  • Raster catalogs per dataset
  • Tile-based storage (configurable resolution)
  • Point value extraction (ST_Value)
  • Band statistics & zoom-level pyramids
  • Configurable pixel type & SRID

📐 Topology Rules (32)

  • Full Esri-equivalent topology rule set
  • Polygon: overlap, gap, coverage, containment
  • Line: dangle, pseudonode, self-intersect, single-part
  • Point: coverage, boundary, coincident detection
  • General: validity, extent, vertex count

🏷️ Domains & Attribute Rules

  • Coded-value and range domains
  • Subtypes with default values & domain assignments
  • Attribute rules: constraint, calculation, validation
  • Trigger events: insert, update, delete
  • Expression-based rule engine

🔗 Relationship Classes

  • Simple & composite relationships
  • 1:1, 1:M, M:N cardinality
  • Forward/backward label navigation
  • Cross-dataset feature linking
  • Typed relationship records

🎨 Cartographic Representations

  • Symbology rules with scale ranges
  • Expression-based feature filters
  • Label rules with font & placement config
  • Priority-based rule ordering
  • JSON-based symbol definitions

Architecture

ptolemy-core

  • Domain types: Dataset, Branch, Changeset, Feature, Diff
  • Serialization with serde
  • Zero external runtime deps

ptolemy-storage

  • Abstract Store trait
  • PostgreSQL/PostGIS implementation
  • Pluggable backend architecture

ptolemy-api

  • Axum 0.8 HTTP router
  • Tower middleware (tracing, CORS)
  • Typed JSON responses

ptolemy-cli

  • Clap-based CLI
  • ptolemy serve --bind 0.0.0.0:3000
  • Future: import, export, branch commands

Quick Start

# Clone and build
git clone https://github.com/TileTopia-HQ/ptolemy.git
cd ptolemy
cargo build --release

# Start with a PostGIS database
export DATABASE_URL="postgres://user:pass@localhost/ptolemy"
./target/release/ptolemy serve --bind 0.0.0.0:3000

# Check health
curl http://localhost:3000/health
